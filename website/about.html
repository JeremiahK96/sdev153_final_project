About Me

My programming journey has been an unusual one; however, it has been a lot of fun and caused me to grow into the software developer that I am today.

The Early Days

When I was 9, I had a computer-tutor toy from 1988 called the VTech Pre-Computer 1000. Among its few features was a simplified BASIC interpreter. You had 2KB of ROM to work with, and you couldn't save your programs, but instead had to type them by hand every time. There was a user manual, which I read, but I mostly just learned by doing. A surprisingly large variety of programs could be written within this tiny sandbox environment! I taught myself the basics of simple data types like strings and arrays, and came up with the Fisher-Yates shuffle on my own.

I first gained access to the Internet in my teens, and I briefly dabbled with Python, HTML, and C++. However, it was always just a hobby. I wanted to pursue 3D computer graphics art, and I never really got serious about programming until my early 20's.

Discovering AtariAge

At some point, I discovered the AtariAge website, specifically the Atari 2600 Programming Forum. This piqued my interest because I had played several Atari 2600 games as a kid. I decided to try writing a program for the Atari, and learned that the programs must be coded in 6502 Assembly Language. I spent a couple days learning 6502 Assembly from an interactive website called Easy 6502, and began writing simple programs for the Atari, testing them out in the Stella emulator.

The Atari 2600 is a difficult beast to tame, and has a lot of quirks. There are 128 bytes of RAM, and there is no video buffer. All you have to work with is a few dozen registers which can be written to which affect the picture and sound. This is both a curse and a blessing, because although even the simplest display kernels can be quite complicated to write, you have a great deal of control over the hardware. This allows you to pull off some fancy tricks, some of which were not even intended by the hardware designers.

Writing a Complete Game for the Atari 2600

I spent a few years tinkering with the Atari in my spare time, and eventually felt competent enough to write a complete game. One day when I checked the forums, I saw Jeff Haber had posted a new display kernel for an 8x8 grid of individually colored tiles. I contacted him, and we agreed to work together on writing a clone of Bejeweled using this kernel.

From June through December of 2020, we wrote the entire game, fitting it into a 2KB ROM. The final bugfixes were finished on January 15, 2021, and the game was later published in cartridge form by AtariAge under the name BLOCKS.

Planned Personal Projects

I have been working on a couple projects since BLOCKS. One project I have been planning for some time is a multi-threaded CD-ripping utility written in C which would allow simultaneous rips using multiple optical drives, all with automatic disc lookups for the metadata. Another project I have been working on recently is a static website generator, which will be written in UNIX shell script as a prototype, but potentially in C or C++ later to maximize the speed of website generation.
