Modularity in Programming

In Python, and in most other programming languages, it is important to make your programs in a modular fashion. You want to break the entire task you want to accomplish down into smaller and smaller steps, so that each step is fairly small and is easy to understand. Modularity allows you to make your code abstract, so that anyone who understands Python can look at it and quickly see what's going on, without having to know all the details.

Simple Example

In this simple example, we ask the user to input a temperature in Celsius, convert the value into Fahrenheit, and then print the result. This is three steps, so we will create and call three functions to do it.

Writing the main() Function

def main():
  celsius = getCelsius()
  fahrenheit = toFahrenheit(celsius)
  showResult(celsius, fahrenheit)

Already, our main function is easy to understand. When you look at this function, it reads like English: Get the Celsius value and store it. Convert it to a Fahrenheit value, and store that. Show both values to the user. Of course, this is an extremely simple example, but it easily shows the power of using functions.

Adding the Functions

Any Python coder who looks at the above code will likely already know exactly what's going on under the hood, but they can always look deeper into each function to find out if they have to.

def getCelsius():
    temp = input("Enter a temperature in Celsius: ")
    return float(temp)

def toFahrenheit(c):
    return 9 / 5 * c + 32

def showResult(celsius, fahrenheit):
  print(celsius, "degrees Celsius is", fahrenheit, "degrees Fahrenheit.")
